Writing

Writing style

Pressgang is dead

Google Documentation guide?

  • Active voice
  • Second person
  • Present tense
  • Clause order

Writing style
compatibility with product docs

  • Titles Should Use a Capital for Almost Every Word
  • Avoid mentioning "Drools", "OptaPlanner", ...
    because in product docs that would be "Decision Manager", "Business Optimizer", ...
    • Use an asciidoc attribute {drools}?

Know your asciidoc

Asciidoc quick reference

  • Use only NOTE, WARNING and IMPORTANT
  • Highlight source code
  • Use internal cross references

Images

Put images in the same directory as the section adoc file.

Unless they are directly used by multiple sections (logo, icons)

Avoid reusing images, reuse sections instead.

How to create diagrams

  • Learn to use Inkscape
  • Set color palette to "Tango Icons"
  • Create images of size 800*600 at 96dpi
  • Expert to png
  • Reuse in documentation and in slides
